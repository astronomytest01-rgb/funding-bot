## Назначение

Бот анализирует исторические ставки фандинга, ищет монеты для LONG/SHORT сбора фандинга, показывает историю ставок, считает доход, ищет дельта-нейтральные пары и добавляет Gemini AI как риск-фильтр.

Критично: API бирж, CoinW/Supabase и функция `analyze_rates` взяты из рабочей Claude-версии и являются эталоном. Gemini не меняет расчёт фандинга.

## Фильтры анализа

Основной фильтр `analyze_rates`:

- LONG: стабильные отрицательные ставки или сильный `neg_avg`.
- SHORT: стабильные положительные ставки или сильный `pos_avg`.
- `full`: проходит стабильность по outlier `<= 25%`.
- `partial`: средняя ставка сильная, но стабильность хуже.
- `fail`: не проходит условия.

Дополнительный фильтр актуальности тренда:

- после основного фильтра бот проверяет последние 4 ставки;
- для LONG минимум половина последних ставок должна быть отрицательной;
- для SHORT минимум половина последних ставок должна быть положительной;
- если тренд развернулся к нулю или в другую сторону, монета отбрасывается.

Этот фильтр применяется в `/filter`, `/analyze` и вечернем отчёте.

## Gemini AI

Gemini используется только после фандинг-фильтров:

- `/ai` анализирует фундаментал, инфраструктуру, ликвидность, волатильность, риск движения 30%+ за сутки и риск скама;
- `/analyze` после обычного отчёта отдаёт найденные монеты в Gemini и показывает только те, которые AI оставил для ручной проверки;
- вечерний отчёт показывает Gemini как рекомендацию по FULL-монетам, но не удаляет монеты из отчёта.

## Gemini prompt contract

Gemini не принимает торговое решение и не пересчитывает funding. Его роль — риск-фильтр по активу после того, как монета уже прошла математические фильтры бота.

Bulk prompt для `/analyze` и вечернего отчёта требует:

- отсеять мемкоины, сверхволатильные щиткоины, слабую инфраструктуру и scam-risk;
- оставить только монеты, пригодные для ручной проверки;
- выводить только список оставленных монет;
- формат строки: `[ЭМОДЖИ] [МОНЕТА] — [причина 3-6 слов]`;
- строго использовать 🟢 для LONG и 🔴 для SHORT;
- если подходящих нет, вернуть `Подходящих фундаментальных монет нет`.

В коде есть дополнительная защита: `enforce_direction_emojis()` исправляет emoji в ответе Gemini по направлению из фандинг-фильтра.

Single prompt для `/ai` анализирует только сам актив: фундаментал, инфраструктуру, ликвидность, волатильность, риск движений 30%+ за сутки и scam-risk. Funding-ставки команда `/ai` не анализирует.

## Команды бота

```text
filter - Анализ монет по фильтрам фандинга
funding - Ставки фандинга по монете
calculator - Калькулятор дохода от фандинга
analyze - Скан рынка + Gemini AI фильтр
ai - Фундаментальный AI-анализ монеты
findpair - Дельта-нейтраль: найти пару лонг+шорт
report - Ручной запуск вечернего отчёта
instruction - Инструкция входа и риск-проверок
settings - Настройки и управление биржами
help - Справка
```

`/filter`:

- пошагово: монеты -> период -> мультивыбор бирж;
- быстрый ввод: `/filter ENJ coinw 7`.

`/funding`:

- история ставок;
- периоды: 1, 3, 7, 14 дней или ручной ввод;
- мультивыбор бирж.

`/calculator`:

- монета -> сумма -> период -> биржи;
- пресеты суммы: 15000, 20000, 25000.

`/analyze`:

- биржа -> метод -> период;
- методы: `Средняя ставка`, `Средний доход`;
- метод `Средний доход` ищет обе стороны: LONG на устойчиво отрицательном funding и SHORT на устойчиво положительном funding;
- после результата запускается Gemini-фильтр, если есть `GEMINI_API_KEY`;
- в Gemini-результате бот закрепляет emoji по направлению: 🟢 LONG, 🔴 SHORT.

`/ai`:

```text
/ai SOL
/ai SOL ENJ RON
```

`/findpair`:

```text
/findpair ENJ
/findpair ENJ RON 14
```

`/report`:

- запускает вечерний отчёт вручную в текущий чат;
- использует ту же логику, что авто-отчёт в 20:00 Europe/Kyiv.

`/instruction`:

- отправляет инструкцию входа и риск-проверок без запуска отчёта;
- первый пункт инструкции: OI на конкретной бирже строго от $1 млн; ордер не должен превышать 1–1.5% от всего OI.

`/settings`:

- inline-кнопки включения/выключения активных бирж;
- состояние не сохраняется после перезапуска.

## Telegram UX и кнопки

Кнопочная логика должна сохраняться как в эталонной Claude-версии. При изменениях нельзя ломать пошаговые сценарии.

- `/filter`: ввод монет -> период `7 дней`, `14 дней`, `Другой период`, `Отмена` -> мультивыбор бирж.
- `/funding`: ввод монеты -> период `1 день`, `3 дня`, `7 дней`, `14 дней`, `Другой период`, `Отмена` -> мультивыбор бирж.
- `/calculator`: ввод монеты -> сумма `15000`, `20000`, `25000`, `Другое`, `Отмена` -> период -> биржи.
- `/analyze`: выбор биржи -> метод `Средняя ставка` или `Средний доход` -> период -> scan progress -> итог -> Gemini-фильтр.
- `/settings`: inline-переключатели активных бирж + `Все ВКЛ/ВЫКЛ`.

Тексты Telegram должны оставаться понятными, с emoji и Markdown-форматированием там, где это уже используется.

## Вечерний отчёт

Если задан `REPORT_CHAT_ID`, бот каждый день в 20:00 Europe/Kyiv запускает job queue задачу. На сервере время указано как 17:00 UTC.

Логика:

1. Сканирует активные биржи.
2. Берёт только `full`-монеты.
3. Применяет фильтр последних 4 ставок.
4. Подбирает дельта-нейтральную пару.
5. Показывает оценку прибыли за 1 день.
6. Отдельным сообщением показывает Gemini-рекомендацию, не удаляя монеты из отчёта.
7. Отдельным сообщением отправляет инструкцию входа и риск-проверок.

## QA перед деплоем

Перед финальным deploy нужно проверить:

1. `python3 -m py_compile bot.py config.py analysis.py ai.py exchanges.py reports.py`.
2. Импорт всех модулей без `NameError`.
3. `/funding` получает данные хотя бы по одной рабочей бирже.
4. `/analyze` в режиме `Средняя ставка` показывает LONG и SHORT группы, если они есть.
5. `/analyze` в режиме `Средний доход` ищет и 🟢 LONG, и 🔴 SHORT.
6. Trend-фильтр работает по последним 4 ставкам: минимум 2 из 4 в нужную сторону.
7. Gemini output сохраняет emoji направления: 🟢 LONG, 🔴 SHORT.
8. Вечерний отчёт использует только `full`-монеты, Gemini показывает как рекомендацию, считает прибыль за 1 день и отправляет инструкцию входа.
9. Оба `.md` файла обновлены перед push.

## Ограничения

1. `/settings` хранит состояние только в памяти.
2. CoinW зависит от Supabase collector.
3. Gemini может ошибаться; это только риск-фильтр.
4. Вечерний скан может занимать несколько минут.
5. Telegram режет сообщения длиннее примерно 4096 символов, бот отправляет чанки.
